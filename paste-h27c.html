hotfix27c -- MAIN only

Dialog widget fix -- isVisible() checks + correct child indices

Root cause found:
  • DZ getWidget() returns a shell object even when dialog is NOT on screen
  • No isVisible() check = bot was clicking invisible widgets every tick (dialog never advanced)
  • Wrong child index: child 4 = NPC NAME TEXT (not button). Continue button = child 5
  • Option dialog: child 1 = header title. First real option = child 2
What changed:
  • ALL dialog widgets now require isVisible() = true before clicking
  • NPC dialog: 231,5 (was 231,4 -- that was the name text, not the button)
  • Option dialog: 219,2 for first option (was 219,1 -- that was header)
  • Debug: game chat shows which widget is/isn't visible every 10 ticks so you can see exactly what's happening
Watch game chat for: "NPC dialog visible (231,5)" or "WAIT tick X -- no visible dialog widget found"
